# Spatio Temporal ID JSON データ形式仕様書(v0.0.1)

## 1. 概要

本書は、「空間 ID」および「時空間 ID」を効率的に表現・交換するための JSON データ形式 **spatial-id-json** を定義するものである。

本フォーマットは、空間 ID および時空間 ID とその属性データの配布・参照を目的とし、冗長性を抑えつつ可読性と拡張性を確保することを重視している。

## 2. 用語の定義

本仕様書で使用する主要な用語を以下に定義する。

- **空間 ID（Spatial ID）**
  空間ボクセルを一意に識別する ID。基本形式は `{z}/{f}/{x}/{y}` とする。

- **時空間 ID（Spatio-temporal ID）**
  空間 ID に時間情報を付加した ID。基本形式は `{z}/{f}/{x}/{y}_{i}/{t}` とする。

- **z（Zoom Level）**
  空間 ID におけるズームレベルを示す整数値。

- **f（Altitude Index）**
  空間 ID における高さ方向（鉛直方向）のインデックス。

- **x（Longitude Index）**
  空間 ID における経度方向（東西方向）のインデックス。

- **y（Latitude Index）**
  空間 ID における緯度方向（南北方向）のインデックス。

- **i（Time Interval）**
  時空間 ID における時間間隔を秒単位で表した値。

- **t（Time Index）**
  時空間 ID における指定された時間間隔に基づく時間インデックス。

## 3. データ構造（Data Structure）

ルートオブジェクトは、以下の 3 つの主要プロパティを必ず持つものとする。

```json
{
  "meta": { ... },
  "fields": [ ... ],
  "ids": [ ... ]
}
```

---

### 3.1. meta オブジェクト

`meta` は、データセット全体に関するメタ情報を保持するオブジェクトである。

| プロパティ  | 必須 | 型      | 説明                                                                                                            |
| ----------- | ---- | ------- | --------------------------------------------------------------------------------------------------------------- |
| version     | 必須 | String  | フォーマットのバージョン（例: `"0.0.1"`）                                                                       |
| description | 推奨 | String  | データセットの説明                                                                                              |
| unique      | 必須 | Boolean | データセット内で時空間 ID の物理的範囲が一意かどうかを示すフラグ。`true` の場合、重複範囲が存在しないことを示す |

---

### 3.2. fields 配列

`fields` は、属性データの定義と値を保持する配列である。

- 配列の各要素は、以下のプロパティを持つオブジェクトとする。

| プロパティ | 必須 | 型     | 説明                               |
| ---------- | ---- | ------ | ---------------------------------- |
| name       | 必須 | String | データ名（例: `"天気"`, `"気温"`） |
| data       | 必須 | Array  | 実際の値を格納する配列             |

- `data` 配列内の要素の型はすべて同一でなければならない。
- 要素の型はプリミティブ型を推奨するが、Object 型も許容される（例: 建物の詳細情報）。

### 3.3. ids 配列

`ids` は、時空間 ID と、それに対応する `fields` への参照を保持する配列である。

データ量削減を目的として、`ids` は階層構造を持つ。

#### 3.3.1. ids 配列の要素（グループ）

各要素は以下のプロパティを持つ。

| プロパティ | 必須 | 型     | 説明                                                                       |
| ---------- | ---- | ------ | -------------------------------------------------------------------------- |
| z          | 必須 | Number | ズームレベル。グループ内すべての ID に適用される                           |
| i          | 必須 | Number | 時間間隔（秒）。時空間 ID を扱う場合に指定し、グループ内すべてに適用される |
| dims       | 必須 | Array  | 次元定義およびデータ参照を持つオブジェクトの配列                           |

#### 3.3.2. dims オブジェクト

`dims` 配列内の各オブジェクトは、具体的な座標範囲と属性データ参照を定義する。

##### 次元プロパティ（`f`, `x`, `y`, `t`）

- 各プロパティの値は以下のいずれかの形式を取る。

  - `[v]`：単一値を示す
  - `[start, end]`：`start` から `end` までの連続範囲（閉区間）を示す

- プロパティが省略された場合は、当該次元の「すべての値」を意味する。

  - `t` を省略した場合：定義されたすべての時間に有効
  - `f` を省略した場合：高さ情報を持たない 2 次元データとして扱う
  - `x` を省略した場合：経度方向の情報を持たないデータとして扱う
  - `y` を省略した場合：緯度方向の情報を持たないデータとして扱う

##### ref プロパティ

- **ref**（必須, Array）は、`fields` 配列への参照リストを示す。

形式：

```json
[ [field_index, data_index], ... ]
```

- `field_index`：`fields` 配列のインデックス（0 始まり）
- `data_index`：該当する `data` 配列のインデックス（0 始まり）
- 複数の属性を同一ボクセルに同時に紐付けることができる
- 配列内の要素が空の場合は、そこに時空間 ID が存在することのみを示す

## 4. セマンティクスおよび制約

### 4.1. 空間 ID・時空間 ID の解決

`ids` 配列から完全な空間 ID または時空間 ID を復元する際は、以下の手順に従う。

1. 親オブジェクトから `z` を取得する。
2. `dims` オブジェクトから `x`, `y`, `f` を取得する。範囲指定の場合は展開する。
3. 時空間 ID の場合は、親オブジェクトの `i` と `dims` 内の `t` を使用する。

### 4.2. データ参照の意味

`ref` は、特定の時空間ボクセルに関連付けられた属性情報を示す。

- 例：

  ```json
  "ref": [[0, 1]]
  ```

  は、`fields[0]`（例: 天気）の `data[1]`（例: 雨）が該当ボクセルの属性であることを示す。

- 空間ボクセルと地物の関係が多対 1 となる場合は、複数の空間 ID が同一の `ref` を持つことで表現する。

## 5. 例

以下は、東京都エリアにおける天気・気温・建物情報を表現した spatial-id-json の例である。

```json
{
  "meta": {
    "version": "0.0.1",
    "description": "東京エリア天気・建物情報",
    "unique": true
  },
  "fields": [
    {
      "name": "天気",
      "data": ["晴れ", "雨", "曇り"]
    },
    {
      "name": "気温",
      "data": [30.3, 28.5]
    },
    {
      "name": "建物情報",
      "data": [
        { "名前": "商社A", "坪単価": 300000, "建物分類": "商用建物" },
        { "名前": "商社B", "坪単価": 30000, "建物分類": "商用建物" }
      ]
    }
  ],
  "ids": [
    {
      "z": 18,
      "i": 3600,
      "dims": [
        {
          "f": [7],
          "x": [232846],
          "y": [103220, 103225],
          "t": [3, 4],
          "ref": [
            [0, 0],
            [1, 0],
            [2, 0]
          ]
        },
        {
          "f": [7],
          "x": [232846],
          "y": [103226],
          "ref": [
            [0, 1],
            [1, 0],
            [2, 1]
          ]
        }
      ]
    }
  ]
}
```
