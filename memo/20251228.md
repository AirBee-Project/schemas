持ち帰りになった時間条件に関する検討(齋藤)

- 時間条件に関する検討
- 完全な開区間
- 半開半閉区間は表現しない
- Value の効率的な参照
- i について考える

Json では表現できないが、作りたい制約

```json
{
  "meta": { "version": "1.0", "description": "東京エリア天気" },
  "values": [
    {
      "name": "天気",
      "data": ["晴れ", "雨", "曇り"]
    },
    {
      "name": "気温",
      "data": [30.3]
    },
    {
      "name": "建物情報",
      "data": [
        { "名前": "商社A", "時価": 300000,"建物分類":"商用建物" },
        { "名前": "商社B", "時価": 30000,"建物分類":"商用建物" }
      ]
    }
  ],
  "ids": [
    {
      //  粒度を外側に持ってくる
      "z": 18,
      "i": 1,
      // それ以外の次元の要素を参照できるようにする
      "dims": [
        {
          "f": [7],
          "x": [232846],
          "y": [103220, 232846],
          "t": [3, 4],

          //Valueの参照
          "value": [
            [0, 0],
            [1, 0]
            [2, 0]
          ]
        },
        {
          "f": [7],
          "x": [232846],
          "y": [103220, 232846],

          //Valueの参照
          "value": [
            [0, 1],
            [1, 0]
          ]
        },
        {
          "f": [7],
          "x": [232846],
          "y": [103220, 232846],

          //Valueの参照
          "value": [
            [0, 1],
            [0, 0]
            [2, 1]
          ]
        }
      ]
    }
  ]
}
```

バイナリデータなどを定義したいので、Json ではなく、`.proto`で定義して、いろいろなところに使いまわせばいいのではないか？
Json で表現する以上、カラム部分が動的型付けなので、非常に不便
