# meta

このデータに関するメタ情報

## version

spatial-id-json というこのデータ形式の標準化バージョン

## description

説明文

## unique

データ形式内の時空間 ID が指し示す範囲が一意であるか

# fields

時空間 ID が持つ値の一覧を保持する部分

- 配列内にはその値の名前と、データ配列がある。
- データ配列の中の型は同じでなければならない
- プリミティブ型が推奨だが、Object も入れられる

# ids

時空間 ID を保持する部分

- i と z は多くの場合、データ内で共通であるため外側で定義する
- dims 内で次元を定義する。
- [number]か[number,number]で次元を定義する。二つの場合は範囲を表現している。
- フィールドが存在しない次元は、その次元の全ての値という意味である。
- `t`がなければ、その値は全ての時間において有効
- `f`がなければ、その値は全ての高度において有効
- `ref`は値の参照。[fiels 配列内の位置、data 配列内の位置]を表す。

```json
{
  "meta": { "version": "0.0.1", "description": "東京エリア天気","unique": true },
  "fields": [
    {
      "name": "天気",
      "data": ["晴れ", "雨", "曇り"]
    },
    {
      "name": "気温",
      "data": [30.3]
    },
    {
      "name": "建物情報",
      "data": [
        { "名前": "商社A", "坪単価": 300000, "建物分類":"商用建物" },
        { "名前": "商社B", "坪単価": 30000, "建物分類":"商用建物" }
      ]
    }
  ],
  "ids": [
    {
      "z": 18,
      "i": 1,
      "dims": [
        {
          "f": [7],
          "x": [232846],
          "y": [103220, 232846],
          "t": [3, 4],
          "ref": [
            [0, 0],
            [1, 0]
            [2, 0]
          ]
        },
        {
          "f": [7],
          "x": [232846],
          "y": [103220, 232846],
          "ref": [
            [0, 1],
            [1, 0]
          ]
        },
        {
          "f": [7],
          "x": [232846],
          "y": [103220, 232846],
          "ref": [
            [0, 1],
            [0, 0]
            [2, 1]
          ]
        }
      ]
    }
  ]
}
```
